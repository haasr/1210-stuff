What Makes the Web the Web

Hyperlinks Changed Everything

One of the things that makes the web unique is hyperlinks. They changed the way consumers interact with content ‚Äî no longer a linear, start-to-finish activity.

Before Links

  • Content consumed top-to-bottom
  • Linear, start-to-finish
  • Like reading a book
  • No jumping between sources

With Hyperlinks

  • Users follow a "breadcrumb trail"
  • Jump site-to-site, or within a site
  • Find exactly the content they want
  • Non-linear exploration
The Anchor Element

The element that creates a hyperlink:

<a>

Whatever appears between <a> and </a> becomes a clickable link.

üé® Default Browser Styling

By default, links are displayed blue and underlined. CSS can override this.

Where to Go ‚Äî href

href = "hypertext reference"

An HTML-speak way of saying "address" ‚Äî tells the browser where to go when the user clicks the link.

href Values

  • Absolute URLs (other sites)
  • Relative URLs (your own site)
  • Page fragments (#section-id)
  • Other protocols (mailto:, ftp:)
Two Types of URLs

üåê Absolute URL

Points to another website. Provides the entire URL ‚Äî a unique, unchanging point on the web.

href="http://www.example.com/default.html"

Easy to spot ‚Äî always contains http:// or https://

üìÅ Relative URL

Points to a file within your own site. The path is relative to the current document's location.

href="default.html" href="#top"

No http:// ‚Äî just a filename or path.

The Rule

Linking within your own site? Use relative URLs.
Linking to another site? Use absolute URLs.
Relative URLs are shorter, portable, and faster (no DNS lookup needed).

The <img> Element

The alt Attribute

Accessibility

  • Screen readers speak the alt text aloud
  • Critical for users with visual impairments
  • Displays when the image fails to load
  • Required for HTML validation

Writing Good Alt Text

  • Describe what the image actually shows
  • Be concise but descriptive
  • Don't start with "image of‚Ä¶" ‚Äî the browser already knows
  • Decorative images: alt="" (empty string)
<!-- Good -->
<img src="campus.jpg" alt="ETSU Sherrod Library with autumn leaves">

<!-- Bad -->
<img src="campus.jpg" alt="image">
<img src="campus.jpg" alt="campus.jpg">
Image Formats

GIF

Graphics Interchange Format

  • Small file sizes
  • Transparent backgrounds
  • Can be animated
  • Good for logos, icons, cartoons
  • Only 256 colors
  • Not good for photos

JPG / JPEG

Joint Photographic Experts Group

  • 16+ million colors
  • Great for photos
  • Lossy compression ‚Üí small files
  • No transparent backgrounds
  • Cannot be animated
  • Not ideal for logos/icons

PNG

Portable Network Graphics

  • 16+ million colors
  • Transparent backgrounds (PNG-24/32)
  • Lossless compression
  • Good for logos, icons, cartoons
  • Larger file sizes than JPG
  • Cannot be animated

The <figure> element wraps an image and its caption together as a semantic unit. Use <figcaption> inside it for the caption text.

Why Use figure?

It's semantic HTML ‚Äî it tells the browser (and screen readers) that the image and caption belong together as a unit. Better than wrapping in a <div>.

Image Links

Using an Image as a Link

Instead of using text for a link's display, you can use an image. Simply nest an <img> inside an <a> tag.

<a href="https://csci1210.com/index.html" target="_blank">
  <img src="images/etsulogo.png" alt="ETSU logo">
</a>

Key Concept: Nesting

  • The <img> is nested inside the <a>
  • The entire image becomes clickable
  • We'll do a lot of nesting in this class
  • Parent ‚Üí child relationship

Best Practices

  • Always include alt text
  • Use target="_blank" for external links
  • Describe where the link goes in the alt text
  • Consider adding a title attribute for clarity
Image Resolution

Resolution Defined

  • Number of pixels per inch (PPI)
  • Web standard: 72 PPI
  • Print standard: 300 PPI
  • Higher PPI = sharper but larger file

Why Not Use 300 PPI for Web?

  • Massive file size impact
  • Screens can't display more than ~96 PPI anyway
  • Wasted bytes = slower page loads
  • File size is the #1 factor in page load time
Two Words:

File Size

Not Everyone Has Fast Internet

  • Satellite connections are slow
  • 3G/4G mobile is still common
  • ~3% of US adults still used dial-up as of 2013
  • File size directly affects user satisfaction
Magic Sizes that Impact Load Speed!

< 14 KB (The "Instant" Zone)

A "magic" rule of thumb, this is the limit for critical, above-the-fold content to load in a single round-trip (TCP slow start). Fitting initial HTML and CSS into this limit makes a site feel instantaneous.

1 MB - 1.5 MB (Ideal Range)

The optimal size for a balance between rich content and speed

< 3 MB (MAX Recommended)

Going over 3 MB typically results in a slow, frustrating experience, particularly on mobile networks, leading to higher bounce rates.

Key Optimization Targets

  • Images: Keep total image weight under 500‚ÄØKB, with individual images ideally under 100‚Äì200‚ÄØKB.
  • Layout: Limit to 50‚Äì60 "blocks" of content to ensure rapid rendering.
  • Code: Minify and compress CSS, JavaScript, and HTML, which can reduce file sizes by up to 60‚Äì90‚ÄØ%.
  • Assets: Use lazy loading to defer loading of images, videos, and scripts until they are needed.

The Browser Cache

What is Caching?

The browser stores a local copy of recently downloaded files. If you revisit a site, it doesn't have to download everything again ‚Äî it loads from the cache instead.

Cache Benefits

  • Dramatically speeds up repeat visits
  • Accessing local memory is very fast
  • Reduces server bandwidth usage
  • Better experience on slow connections

Optimizing Images for the Web

What Does "Optimize" Mean?

Edit image files so the resulting file size is as small as possible without unacceptably losing quality. Three main techniques:

Cropping

Remove unimportant parts of the image. Only show what's relevant. Can dramatically reduce file size.

Resizing

Reduce dimensions and/or resolution. A 1920√ó1080 photo doesn't need to be displayed full-res on a 400px column.

Compression

Increase JPEG compression ratio. Adjust the quality slider. A quality of 75‚Äì85 is often indistinguishable from 100 at a fraction of the size.

552 KB ‚Üí 24 KB

Real example: by adjusting resolution, dimensions & compression ‚Äî 4.3% of original file size, still looks great on a web page!

Targeted Site Users

Who Are You Building For?

Every well-designed site has a targeted set of users. You cannot create a site that appeals to everyone ‚Äî and trying to do so results in a site that appeals to no one.

üë• Why Define Users?

  • Profile users based on site mission & goals
  • Define groups to focus the design
  • Know what content and design will appeal
  • Know what to leave out

üìä Where Information Comes From

  • Client's marketing department
  • Sales/marketing employees
  • Existing website analysis
  • Surveys & customer research
User Personas

What is a Persona?

A persona is a description of a hypothetical site user ‚Äî their interests, goals, technology comfort, and concerns. We create one per targeted user group to guide design decisions.

‚ùì Questions to Answer

  • Who will visit this site?
  • What do they have in common?
  • What are their interests & goals?
  • What do they like / dislike?
  • What technology do they use?

üìã Focal User Characteristics

  • Age ranges & gender
  • Education level & occupation
  • Computer & web experience level
  • Hobbies and personal goals
  • Devices used (desktop, mobile, etc.)
Example Persona ‚Äî Joan
Joan is a new teacher in her second year, recently certified. She's starting to get the hang of things but realizing how much she needs to improve. She wants to get better, but also wants to be smart about it.
Needs
  • Measurable impact
  • Project that self-manages
Wants
  • Something that doesn't take long to set up
  • Products that integrate with existing tech
Values
  • Student growth
  • Time savings
  • Products that fit her current workflow
Fears
  • Cutting into already sparse free time
  • Starting something unsustainable
  • Doesn't actually improve student outcomes

Requirements Gathering

Understanding what the site must do
Requirements

What are Requirements?

Requirements are the elements that must be present in our final product to achieve success. They come from multiple stakeholders ‚Äî not just the designer.

Designer

Site must have correct utility and be usable.

Site Owner (Customer)

Site must allow the sale of products.

End User

Site must allow me to select products by browsing by color, style, product name, or size.

‚ö†Ô∏è Key Point

Designers do not determine specific requirements. They gather requirements specified by clients and end users. The designer facilitates the process.

Requirements Elicitation

Key Terms

  • Requirements Analysis ‚Äî learning and understanding user needs
  • Requirements Elicitation ‚Äî discovering requirements by communicating with users
  • User Task Analysis ‚Äî determining how requirements should be implemented

Elicitation Techniques

  • Interviews & questionnaires ‚Äî specific, detailed questions
  • Brainstorming sessions ‚Äî group discussion of ideas
  • Storyboards ‚Äî diagram alternatives, discuss pros/cons
  • Role playing ‚Äî walk through a specific requirement
  • Prototyping ‚Äî rough drafts for examination

üí° Important Note

Users generally will not have thought through their requirements in advance. They must be guided in the process of stating them. A client saying "I want an online store" leaves many design decisions open.

Types of Requirements

Specific Requirements

Tied to functional demands or design attributes:

  • "Our company color is royal blue; we'd like that featured in the design"
  • "We'd like site visitors to be able to chat with technical support"

General Requirements

Derived from knowledge of targeted users:

  • Older audience ‚Üí avoid flashy presentation
  • High color contrast for readability
  • Larger text for accessibility
  • Mobile-first if users are on phones

Requirements vs. Mission Statement

Sometimes stated requirements conflict with the site's mission statement. The client/site owner must be made aware of conflicts so they can decide how they'll be handled. The designer facilitates, but doesn't decide.
